Copyright, Creative Common(CC)Licenses, Fair Use

Original work such as artwork, books, music and film etc may display the © symbol. This is to show that ownership of that work has been claimed, however it is not required to display the © symbol. Creative common licenses have been created to grant anyone from individual creators to organisations standardized copyright permissions to their creative work. There are many types of licenses however I will only be going over four as well as fair use.
1. The attribution license: allows others to distribute, remix, adapt and build upon the original work, even commercially. So long as credit is given for the original creation.
2. The NoDerivs License: Allows others to reuse the work for any purpose, however it cannot be adapted and credit must be provided to the original content creator.
3. The NonCommercial License: Allows others to adapt, remix and build upon the original woork non-commercially. although they must aknowledge the original creator and be non-commercial, they do not have to license their derived work on the same terms.
4. The NonCommercial-NoDerivs license: is the strictest of the 6 main licenses. This license only allows others to download the original work and share them as long as they give the original creator credit, however the content cannot be modified nor used commercially If you want to know more about copyright licenses please go to: About The Licenses. New Zealand uses "fair Dealing" which permits a party to use copyrighted material,without the owner's permission, for the purposes of research, private study, etc

How to improve your website SEO

Five Ways to improve your sites ranking (SEO) has tips on improving SEO.
1. Keywords/Keyword Phrases: Identify and target a specific phrases or keywords on your website so that users can search those phrases/keywords to find the webpage.
2. Frequently Update Content: Regularly updated content shows a sites relevancy, this means that search engines are also more likely to show your content if it is updated over old websites that have not been updated.
3. Metadata: Is the textual description that a browser may use in your page search return.
4. Have a link-worth site: A website that is helpful, unbiased, and content rich is most likely to attract links from other websites thus improving search engine optimization Using descriptive links can also add value to readers, including those with disabilities or users using screen readers.
5. Use alt tags: Always describe videos and images with alt tags (alternative descriptions). These allow search engines to locate your page.

Web performance and maintenance

5 Tips for reducing load speed
1. Optimize your images: different file formats have different sizes, if your website is experiencing low load times try changing the file type of some of your images.
<2. Don't scale down your images: if you scale down images within the code of a website, the website will try to scale it down as the page is loading, this will increace your load time as there is extra work being down as the page is being loaded.
3. Put style sheet references at the top: Placing your stylesheet references to the <head> of your HTML document help pages load faster because it allows your pages to render the styles progressively instead of right at the end.
4. Put script references at the bottom: scripts can be long and complicated functions that could take web browsers much longer to load that stylesheets, this can make website load times feel very long. Put script references at the bottom of the HTML document so the website loads the main parts of the website before it tries to load the functions.
5. Use separate javascript and CSS files: IF all the javascript and CSS for your website is within the HTML document, this will make the website load the css and functions every time the website loads. This will make the website feel like it's got a long load time. It is also neater and easier to use external files.
5 things to do when maintaining a website
Make Regular backups of your site, Test Load Speed, Fix dead links and errors, Test Browser Compatibility and check search rankings.

Web Security

5 common cyber attack methods and preventatives:
1. Password attacks: If your website has some sort of authentication then password attacking could become a problem, a preventative for this type of attack is to have an account lockout e.g. lock the account after 5 failed password attempts.
2. DDoS (Denial of Service) attacks: A denial of service attack is aimed at denying anyone but the attacker access to the webpage in order to do some sort of harm to it or even theft. A preventative of this type of attack is to use a proxy like cloudflare where they are able to withstand DDoS attacks because of their 5 second timer. This prevents the DDoS going to the desired website as they begin running as soon as it's started. Cloudflare adds a timer before the website is accessed, this means that the DDoS goes to cloudflare rather than your webserver.
3. Phishing attacks: Phishing attacks are easily missed and easily avoided, however there are those people who miss the clear indicators of the attack and click on a link thinking it's for something else. The only way to prevent this is to regularly quiz people on phishing attacks and inform them on certain bits of content that indicate an attack attempt.
4. Ransomware: Ransomware is everywhere on the internet from dodgey applications to malicious emails, ransomware can be used to hold a system "hostage". Preventatives for this type of attack would be not open suspicious links, download from non reputable sources, make sure backups have been created almost daily if a roll back is needed etc.
5: Cross site scripting : Cross site scripting(XSS) occurs when an untrusted source is allowed to inject their own code into a web application and that dynamic code if includes with content delivered to the victims browser. XSS allows an attacker to steal session cookies, allowing the attacker to pretend to be the user, however if can also be used to spread malware, deface websites and reate havoc on social networks. Preventatives for this type of attack include; filtering ingress data, encoding egress data, and content security policies
